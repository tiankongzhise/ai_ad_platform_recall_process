# 广告商业平台 Recall 服务接口文档（基于当前代码）

本文档以项目当前代码实现为准，覆盖所有对外 HTTP 接口与统一响应规范。

## 1. 基础说明

- Base URL：`http://{host}:{port}`
- 默认端口：`8080`
- 协议：HTTP/JSON
- 鉴权方式：`Authorization: Bearer <JWT>`

## 2. 统一响应结构

所有接口（成功/失败）均返回：

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {}
}
```

字段说明：

| 字段 | 类型 | 含义 |
|---|---|---|
| code | int | 业务状态码，`0` 为成功 |
| message | string | 业务提示信息 |
| data | object/null | 业务数据，失败时通常为 `null` |

常见错误码：

| 错误码 | 含义 |
|---|---|
| 0 | 成功 |
| 1001 | 用户名已存在（也用于手机号已注册） |
| 1002 | 用户名/密码错误、验证码错误等认证失败 |
| 1003 | Token 无效或已过期 |
| 1004 | 未提供 Token |
| 1005 | Token 格式错误 |
| 2001 | 缺少必填参数 |
| 2002 | `state` 参数格式错误 |
| 2003 | 参数解析错误（保留） |
| 3001 | 通知 URL 格式错误 |
| 5001 | 内部错误或请求绑定错误 |

## 3. 认证与账号接口

### 3.1 注册

- 方法/路径：`POST /api/auth/register`
- 鉴权：否

请求体：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| username | string | 是 | 用户名，长度 3~64 |
| password | string | 是 | 密码，最少 8 位 |
| phone | string | 是 | 手机号，固定 11 位 |
| code | string | 是 | 注册验证码（当前 Mock 验证） |

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|
| username | string | 注册用户名 |
| uid | string | 系统生成的 32 位十六进制 UID |
| api_token | string | 系统生成的长期 ApiToken |

### 3.2 登录

- 方法/路径：`POST /api/auth/login`
- 鉴权：否

请求体：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|
| token | string | JWT（兼容旧接口字段名） |
| expires_at | string(datetime) | JWT 过期时间 |
| username | string | 用户名 |
| uid | string | 用户 UID |

### 3.3 使用 ApiToken 换取 JWT/RefreshToken

- 方法/路径：`POST /api/auth/create-jwt-token`
- 鉴权：否

请求体：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| api_token | string | 是 | 用户长期 ApiToken |

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|
| jwt_token | string | JWT |
| refresh_token | string | RefreshToken |
| expires_at | string(datetime) | JWT 过期时间 |
| refresh_expires_at | string(datetime) | RefreshToken 过期时间 |

### 3.4 使用 RefreshToken 刷新 JWT

- 方法/路径：`POST /api/auth/refresh-jwt-by-refresh-token`
- 鉴权：否

请求体：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| refresh_token | string | 是 | RefreshToken |

返回 `data` 同 3.3。

### 3.5 使用 ApiToken 刷新 JWT

- 方法/路径：`POST /api/auth/refresh-jwt-by-api-token`
- 鉴权：否

请求体：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| api_token | string | 是 | 用户长期 ApiToken |

返回 `data` 同 3.3。

### 3.6 注销

- 方法/路径：`POST /api/auth/logout`
- 鉴权：是
- 请求体：无
- 返回：`data=null`

### 3.7 刷新当前 JWT（旧兼容接口）

- 方法/路径：`POST /api/auth/refresh`
- 鉴权：是
- 请求体：无

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|
| token | string | 新 JWT |
| expires_at | string(datetime) | 新过期时间 |

### 3.8 发送注册验证码

- 方法/路径：`POST /api/auth/send-code`
- 鉴权：否

请求体：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| phone | string | 是 | 手机号，11 位 |
| username | string | 否 | 预留字段，当前可不传 |

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|
| message | string | 一般为“验证码已发送” |

### 3.9 发送重置密码验证码

- 方法/路径：`POST /api/auth/send-reset-code`
- 鉴权：否

请求体：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| phone | string | 是 | 手机号，11 位 |
| username | string | 否 | 传入后会校验用户名与手机号匹配 |

### 3.10 重置密码

- 方法/路径：`POST /api/auth/reset-password`
- 鉴权：否

请求体：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| username | string | 是 | 用户名 |

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|

### 3.11 通过用户名查询 UID（含已注销）

- 方法/路径：`GET /api/auth/get-uid`
- 鉴权：否

Query 参数：

| 参数 | 类型 | 必填 | 含义 |

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|

### 3.12 通过用户名查询活跃 UID

- 方法/路径：`GET /api/auth/get-activate-uid`
- 鉴权：否
- Query 参数与返回结构同 3.11，仅查询未注销账户。

### 3.13 获取账户信息

- 方法/路径：`POST /api/account/update-api-token`
- 鉴权：是

请求体：

| 字段 | 类型 | 含义 |
|---|---|---|

| jwt_ttl_hours | int | JWT 配置有效期（小时） |
| refresh_ttl_hours | int | RefreshToken 配置有效期（小时） |

## 4. Recall 回调与查询接口

### 4.1 回调入口

- 方法/路径：`GET /recall`
- 鉴权：否

Query 参数：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| state | string | 否 | 回调核心参数，格式 `uid_platform_user_tag` |
| 其他任意参数 | string | 否 | 会被原样记录到 `params` 字段 |

`state` 子字段规则（由服务端解析）：

| 子字段 | 类型 | 规则 | 含义 |
|---|---|---|---|
| uid | string | 32 位十六进制 | 用户 UID |
| platform | string | 1~13 位数字 | 平台编码 |
| user_tag | string | 1~13 位数字 | 用户标签 |

返回说明：

- 当不传 `state`：返回接口说明 JSON（`code=0`）
- 当传了合法 `state`：保存回调并返回 `data={"success": true}`
- 当格式非法：返回 `code=2002`

### 4.2 分页查询

- 方法/路径：`GET /api/query`
- 鉴权：是

Query 参数：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| platform | string | 否 | 平台编码 |
| user_tag | string | 否 | 用户标签 |
| user_name | string | 否 | 兼容旧参数名，等价 `user_tag` |
| page | int | 否 | 页码，默认 1 |
| page_size | int | 否 | 每页条数，默认 20，最大 100 |

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|
| total | int64 | 符合条件总记录数 |
| page | int | 当前页 |
| page_size | int | 当前页大小 |
| records | array<object> | 回调记录列表 |

`records` 每项字段：

| 字段 | 类型 | 含义 |
|---|---|---|
| user_name | string | 归属用户名 |
| uid | string | 归属用户 UID |
| platform | string | 平台编码 |
| user_tag | string | 用户标签 |
| params | string | JSON 字符串，包含 state 字段和其他 query 参数 |
| created_at | string(datetime) | 创建时间 |

### 4.3 查询最新一条

- 方法/路径：`GET /api/query/latest`
- 鉴权：是
- Query 参数同 4.2（不需要分页参数）
- 返回 `data`：单条回调记录对象（字段同 `records` 单项）

### 4.4 查询历史（最多 1000 条）

- 方法/路径：`GET /api/history`
- 鉴权：是
- Query 参数同 4.2（不需要分页参数）

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|
| total | int | 返回条数 |
| records | array<object> | 回调记录列表（最多 1000 条） |

## 5. 通知接口

### 5.1 设置通知地址

- 方法/路径：`POST /api/notify/set`
- 鉴权：是

请求体：

| 参数 | 类型 | 必填 | 含义 |
|---|---|---|---|
| notify_url | string | 是 | 接收通知的 HTTP/HTTPS 地址 |

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|
| notify_url | string | 已保存的通知地址 |

### 5.2 获取通知地址

- 方法/路径：`GET /api/notify/get`
- 鉴权：是

返回 `data`：

| 字段 | 类型 | 含义 |
|---|---|---|
| notify_url | string | 当前用户通知地址（未设置则为空字符串） |

### 5.3 出站通知（系统回调到你的服务）

当 `/recall` 成功处理且用户配置了 `notify_url` 后，系统会异步 POST：

- 方法：`POST {notify_url}`
- Header：`Content-Type: application/json`
- Body：

```json
{
  "user_name": "alice",
  "platform": "1001",
  "user_tag": "2002"
}
```

字段说明：

| 字段 | 类型 | 含义 |
|---|---|---|
| user_name | string | 回调归属用户名 |
| platform | string | 平台编码 |
| user_tag | string | 用户标签 |

重试策略（来自配置）：

- 超时：`notify.timeout_seconds`
- 重试次数：`notify.retry_times`
- 重试间隔：`notify.retry_interval_seconds`
