# 广告商业平台 Recall 回调处理系统 技术文档（实现对齐版）

## 1. 系统概览

本系统用于接收广告 Recall 回调并存储到数据库，提供基于登录用户的数据查询能力，并在回调成功后向用户配置的通知地址异步推送消息。

技术栈：

- Go + Gin
- GORM（MySQL / PostgreSQL）
- JWT（HS256）
- 静态前端页面（`/web`）

## 2. 代码结构

## 3. 核心模型

### 3.1 用户 `users`

- `user_name`: 用户名
- `uid`: 32 位十六进制用户唯一标识
- `phone`: 手机号
- `password`: bcrypt 哈希
- `api_token`: 长期凭证
- `notify_url`: 回调通知地址
- `status`: 1 正常 / 0 已注销
- `logout_at`: `-1` 为活跃，其他为注销时间戳

### 3.2 JWT 记录 `tokens`

- `user_id`
- `token`
- `expires_at`

### 3.3 RefreshToken 记录 `refresh_tokens`

- `user_id`
- `token`
- `expires_at`

### 3.4 回调记录 `recall_records`

- `user_name`
- `uid`
- `platform`
- `user_tag`
- `params`（完整参数 JSON 字符串）
- `created_at`

## 4. 请求链路

### 4.1 鉴权链路

1. 读取 `Authorization` 请求头  
2. 校验 `Bearer <token>` 格式  
3. 解析并验证 JWT 签名与过期时间  
4. 校验用户状态（未注销）  
5. 注入上下文 `user` 和 `token`

### 4.2 回调链路

1. 接收 `GET /recall`
2. 读取 `state=uid_platform_user_tag`
3. 校验 uid/platform/user_tag 格式
4. 通过 uid 反查用户
5. 存储回调记录
6. 异步触发通知（若配置了 `notify_url`）

## 5. 配置说明

配置文件：`config.yaml`

关键配置段：

- `server`: 启动地址与模式
- `database`: 数据库驱动与连接池参数
- `token`: `secret`、`expiry_hours`、`jwt_ttl_hours`、`refresh_ttl_hours`
- `notify`: 超时与重试参数

环境变量覆盖（优先级高于 YAML）：

- `DB_HOST` `DB_PORT` `DB_USER` `DB_PASSWORD` `DB_NAME`
- `TOKEN_SECRET`

## 7. 错误码

| 错误码 | 含义 |
|---|---|
| 0 | 成功 |
| 1001 | 用户名已存在（或手机号已被注册） |
| 1002 | 认证失败（用户名密码错误、验证码错误等） |
| 1003 | Token 无效或过期 |
| 1004 | 缺少 Token |
| 1005 | Token 格式错误 |
| 2001 | 缺少必填参数 |
| 2002 | state 格式错误 |
| 2003 | 参数解析错误 |
| 3001 | 通知 URL 格式错误 |
| 5001 | 内部错误 |

## 8. 已知实现特性

- 验证码为 mock：注册 `930108`，重置 `931216`
- `/api/query*` 接口在后端强制按当前登录用户过滤，避免跨用户查询
- 更换 `ApiToken` 后会清理旧 JWT 和 RefreshToken
